## 1. WHAT WAS BUILT

### 1.1 GitHub Action: NOUS Behavioral Diff

A CI/CD workflow that automatically runs semantic behavioral analysis on every Pull Request that modifies `.nous` files.

**Files created:**

| File | Location | Purpose |
|------|----------|---------|
| `nous-diff.yml` | `.github/workflows/` | GitHub Actions workflow definition |
| `pr_comment_generator.py` | `.github/scripts/` | JSON parser + GitHub API comment poster |

**What it does:**

1. Triggers automatically on any PR that touches a `.nous` file
2. Checks out both the base branch and the PR head branch
3. Uses the GitHub API to identify exactly which `.nous` files changed
4. Runs `python cli.py diff base.nous head.nous --json` on each changed file
5. Parses the JSON output (cost, topology, protocol, findings)
6. Posts a single formatted Markdown comment on the PR with:
   - Overall status badge (🟢 All Clear / 🟡 Warnings / 🔴 Breaking)
   - Aggregated cost impact (e.g., `$145.80/mo → $45.14/mo (-69.0%)`)
   - Per-soul cost breakdown with delta badges
   - Topology changes (souls added/removed/modified)
   - Protocol changes (messages added/removed)
   - Findings table sorted by severity (CRITICAL → WARNING → INFO)
7. Updates the same comment on subsequent pushes (no spam)

**Comment format example:**

```
## 🟡 NOUS Behavioral Diff

> Semantic impact analysis for **1** modified `.nous` file

| | Summary |
|---|---------|
| **Cost Impact** | 🟢 **$145.80/mo → $45.14/mo (-69.0%)** |
| **Findings** | 1 critical · 4 warnings · 4 info |
| **Status** | 🟡 Warnings Detected |

### 📄 `gate_alpha.nous`

| Metric | Before | After | Change |
|--------|--------|-------|--------|
| **Monthly Cost** | `$145.80/mo` | `$45.14/mo` | 🟢 **-69.0%** |

#### Topology
- 🔴 Soul removed: **Monitor**
- 🟡 Soul modified: **Scout**

#### Findings
| | Severity | Code | Detail |
|---|----------|------|--------|
| 🔴 | CRITICAL | `COST` | Total cost change: -69.0% |
| 🟡 | WARNING | `TIER` | Soul Scout: tier changed Tier1 → Groq |

---
🧠 Generated by NOUS Behavioral Diff · docs
```

**Why this approach:**

- `git diff` failed across separate checkout directories (no common git history), so we switched to the GitHub REST API (`/pulls/{number}/files`) which reliably returns the list of changed files regardless of checkout strategy.
- The workflow uses `PYTHONPATH` pointing to the head checkout instead of `pip install` because our project has a flat file layout (no `nous/` package directory). This avoids the need for restructuring 40+ files.
- `GITHUB_TOKEN` is automatically provided by GitHub Actions — no manual secret configuration needed.
- The comment uses an HTML marker (`<!-- nous-behavioral-diff -->`) to find and update itself on subsequent pushes, preventing comment spam.

## 2. ISSUES ENCOUNTERED AND RESOLVED

### 2.1 Git Diff Failure (Exit Code 128)

**Problem:** The initial workflow used `git diff --name-only` to find changed files, but with two separate `actions/checkout` into different directories (`head/` and `base/`), there was no shared git history.

**Fix:** Replaced with GitHub REST API call to `/repos/{owner}/{repo}/pulls/{number}/files` which returns changed files without needing git history.

### 2.2 Node.js 20 Deprecation Warning

**Problem:** `actions/checkout@v4` and `actions/setup-python@v5` run on Node.js 20, which GitHub is deprecating.

**Impact:** Warning only — not a blocker. Will auto-resolve when action maintainers update to Node.js 24.

### 2.3 Externally Managed Python Environment

**Problem:** `pip install -e .` failed on neurodoc because Debian 12 marks the system Python as externally managed (PEP 668).

**Fix:** Used `--break-system-packages` flag. Safe in our case since we control the server entirely.
